# Customer-agnostic connector config keys — the ONLY keys strip_catalog.py keeps.
#
# strip-by-default: a config value survives only if its leaf key name appears
# here. Everything else (hosts, credentials, accounts, database/schema/table
# names, buckets, regions, start dates, cursors, stream names, customer-authored
# reports, ...) is removed.
#
# Add a key here ONLY if it reveals NOTHING about the customer — a pure
# behaviour / format / performance / scheduling toggle. When in doubt, leave it
# out (stripping a harmless knob is safe; keeping a revealing one is a leak).
#
# Only SCALAR-valued keys (and lists of scalars) are kept, matched by their own
# leaf key name. Listing an OBJECT's key has no effect — objects are always
# recursed into. To keep the contents of a nested object, add each scalar key
# inside it individually.
# Deliberately EXCLUDED as too-revealing or too-generic: auth_type, catalog_type,
# compute_type, method, mode, is_sandbox, timezone, region, name, limit, days,
# start_date, cursor*, and every host/credential/account/schema/table field.
#
# Derived from a survey of every Go + Python connector config schema. Both
# camelCase and snake_case spellings are listed because Go and Python differ.
# Re-review when connectors add config fields.
